==============================================================================
1. INTRODUCTION                                             *slidev-introduction*

Slidev.vim adds navigation, ghost-text slide numbers, and editing helpers to
Slidev (https://sli.dev) presentation files.

==============================================================================
2. DETECTION                                                   *slidev-detection*

A buffer is recognised as a Slidev file when all checks up to the configured
strictness level pass:

    Level 1  First line of the file is exactly `---`
    Level 2  A package.json is found somewhere up the directory tree
    Level 3  package.json lists a slidev package as a dependency (default)
    Level 4  A prettier config in the project has an overrides entry whose
             files glob matches *.md / *.markdown

Files whose basename (case-insensitive) appears in |g:slidev_ignored_names|
are always skipped regardless of strictness.

==============================================================================
3. CONFIGURATION                                           *slidev-configuration*

*g:slidev_strictness*
    Detection strictness level. See |slidev-detection|.
    Default: 3

*g:slidev_ignored_names*
    List of lowercase basenames that are never treated as Slidev files.
    Default: readme.md, contributing.md, license.md, licence.md,
             changelog.md, changes.md, history.md, authors.md,
             contributors.md, security.md, support.md,
             code_of_conduct.md, funding.md, governance.md

    Examples: >
        " append to the default list
        g:slidev_ignored_names += ['notes.md', 'todo.md']

        " replace entirely
        g:slidev_ignored_names = ['readme.md']
<

*g:slidev_dev_port*
    Port on which the Slidev dev server listens. Used to build the slide URL
    in Mode A preview (|:SlidevPreviewToggle|).
    Default: 3030

*g:slidev_screenshot_cmd*
    Shell command template used to capture a screenshot of a slide (Mode A).
    Must contain both {url} and {output} placeholders, which are substituted
    at render time with the slide URL and the output file path respectively.
    Example: >
        let g:slidev_screenshot_cmd =
            \ 'brave --headless=new --screenshot={output} --window-size=1920,1080 {url}'
<
    When unset (the default), the plugin falls back to `pnpm slidev export`
    (Mode B).
    Default: '' (empty — Mode B is used)

*g:slidev_screenshot_path*
    File path where Mode A writes the screenshot image and where |chafa| reads
    it from.
    Default: /tmp/slidev-preview/slide.png

==============================================================================
4. MAPPINGS                                                     *slidev-mappings*

All mappings are buffer-local and only active in detected Slidev files.

    <C-n>       Move to the next slide separator (supports count: 5<C-n>)
    <C-p>       Move to the previous slide separator (supports count: 3<C-p>)
    <leader>s   Open the command line pre-filled with :SlidevGoToSlideNum
    <leader>a   Add a new slide after the current one
    <leader>D   Delete the current slide
    <leader>R   Run `pnpm dev %` in a terminal split
    <leader>i   Print slide position and plugin variable values (:SlidevInfo)
    <leader>z   Toggle single-slide focus view (:SlidevFocus)
    <leader>P   Toggle live slide preview (:SlidevPreviewToggle)

In focus mode two extra buffer-local mappings are active:

    k           Move up; when cursor reaches the closed fold above the slide,
                pressing k again exits focus mode.
    j           Move down; same boundary behaviour as k.

==============================================================================
5. COMMANDS                                                     *slidev-commands*

The following commands are always globally available (any buffer):

*:SlidevInfo*
    Print the current slide position (e.g. "slide 2 / 5"), whether the plugin
    is active in the current buffer, all strictness levels with descriptions
    (marking the current one), and the current values of |g:slidev_strictness|
    and |g:slidev_ignored_names|.

*:SlidevEnable*
    Activate Slidev in the current buffer, bypassing automatic detection. Also
    re-enables a buffer that was previously disabled with |:SlidevDisable|.

The following commands are buffer-local and only available after detection
(or after |:SlidevEnable|):

*:SlidevGoToSlideNum* {n}
    Jump to slide number {n} (1-based).

*:SlidevRefresh*
    Manually refresh ghost-text slide number annotations.

*:SlidevFocus*
    Toggle single-slide focus view. When active, all content outside the
    current slide is folded away so only the current slide is visible.
    While focused, <C-n>/<C-p> navigate to the next/previous slide and
    immediately re-apply focus there. Pressing k/j into the closed fold
    above/below the slide and pressing the key once more exits focus mode.
    Run again (or use |zR|) to unfold everything. The previous foldmethod is
    restored when focus is toggled off.

*:SlidevPreviewToggle*
    Toggle a live slide preview in a vertical split to the right. When enabled:
    - Opens a |:terminal| buffer running `chafa` to render the current slide
      as an image directly in the terminal.
    - Refreshes automatically on slide navigation (<C-n>/<C-p>) and on file
      save (|BufWritePost|).
    - Requires `chafa` to be installed and in PATH.

    Two rendering modes are supported:

    Mode A (custom screenshot command): When |g:slidev_screenshot_cmd| is
    configured, that command is invoked to capture the current slide from the
    running dev server. The slide URL is built as
    `http://localhost:{g:slidev_dev_port}/{slidenum}`. Example: >
        let g:slidev_screenshot_cmd =
            \ 'brave --headless=new --screenshot={output} --window-size=1920,1080 {url}'
<
    Mode B (pnpm slidev export): When |g:slidev_screenshot_cmd| is empty (the
    default), the plugin runs `pnpm slidev export --format png --range N` to
    render the current slide. This is slower (Playwright startup) but requires
    no extra tools beyond pnpm.

    Use |:SlidevPreviewRefresh| to force a re-render at any time.

*:SlidevPreviewRefresh*
    Force a re-render of the current slide in the preview split. Has no effect
    when the preview is not active.

*:SlidevDisable*
    Deactivate all Slidev features for the current buffer (mappings, ghost
    text, commands, autocmds). Use |:SlidevEnable| to reactivate.
